Olist E-Commerce Analytics (SQL Project)
Project Overview: -
This project analyzes the Olist Brazilian e-commerce dataset using PostgreSQL to extract business insights from transactional data. The focus is on SQL-based data modeling, KPI Development, and analytics-ready queries commonly used in business intelligence roles.

Dataset: -
~100K orders
~110K order items
~99K customers
~3K sellers
Data includes orders, products, customers, sellers, payments, and delivery timelines.
What I Built: -
Analytics views (order-level, line-level, dimensions, monthly KPIs)

7 portfolio SQL queries covering:- Executive KPIs, Monthly performance trends, Product category revenue analysis, Seller performance, Customer segmentation (RFM), Delivery performance by region and time, Revenue growth and running totals

Key Insights: -
#1. Revenue is highly concentrated in a small number of product categories.
#2. Delivery performance varies significantly by state and month.
#3. High-value customers can be identified using transaction history alone.
#4. Freight costs meaningfully impact seller-level profitability.
SQL Concepts Demonstrated: -
CTEs and subqueries, Aggregations and GROUP BY, Window functions, Date and interval calculations, KPI engineering with CASE logic

Tools Used: -
PostgreSQL, VS Code
